# 📝 YAML Essentials: Complete Guide for DevOps

## 👋 Welcome!

**Starting Point:** Zero YAML knowledge  
**End Goal:** Confident YAML user for DevOps configurations

This guide will take you from never using YAML to writing production-ready configuration files for Docker, Kubernetes, CI/CD pipelines, and more.

---

## 📚 Complete Learning Path

### Phase 1: Foundations (Lessons 1-3)
Master the basics of YAML syntax and structure

- ✅ **01** - [What is YAML?](01-what-is-yaml.md) - Understanding YAML and why it matters
- ✅ **02** - [Basic Syntax](02-basic-syntax.md) - Scalars, indentation, and formatting
- ✅ **03** - [Data Types](03-data-types.md) - Strings, numbers, booleans, null

### Phase 2: Structures (Lessons 4-5)
Working with collections and complex data

- ✅ **04** - [Lists and Arrays](04-lists-arrays.md) - Sequences and collections
- ✅ **05** - [Dictionaries and Maps](05-dictionaries-maps.md) - Key-value pairs and nesting

### Phase 3: Advanced Features (Lessons 6-7)
YAML power features

- ✅ **06** - [Anchors and Aliases](06-anchors-aliases.md) - Reusing configuration with &, *, <<
- ✅ **07** - [Multi-line Strings](07-multiline-strings.md) - |, >, and string formatting

### Phase 4: DevOps Applications (Lessons 8-10)
Real-world YAML in DevOps tools

- ✅ **08** - [Docker Compose](08-docker-compose.md) - Writing docker-compose.yml files
- ✅ **09** - [Kubernetes](09-kubernetes.md) - Pods, Deployments, Services manifests
- ✅ **10** - [CI/CD Pipelines](10-cicd-pipelines.md) - GitHub Actions, GitLab CI, Jenkins

### Phase 5: Reference
- ✅ **11** - [Quick Reference](11-quick-reference.md) - Cheat sheet and best practices

## 📖 How to Use This Guide

1. **Start from the beginning** - Even if you know some YAML, review the basics

4. **Complete the exercises** - Hands-on practice is essential

## 💡 Why YAML?

YAML is the configuration language of choice for:
- 🐳 **Docker Compose** - Multi-container applications
- ☸️ **Kubernetes** - Container orchestration
- 🔄 **CI/CD** - GitHub Actions, GitLab CI, CircleCI
- 🔧 **Ansible** - Infrastructure automation
- ⚙️ **Helm** - Kubernetes package manager
